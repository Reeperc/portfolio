Network Architecture Project

Project Description

  • Set up a network and system infrastructure allowing users to play. The game organizer can create tournaments.
  • Implement network security backups for the configurations of routers/switches, tournament history, and the distribution and segmentation of a Class C IP address into 3 subnets.
  • Players must be able to connect to the network via WiFi, access a website through their session, and receive an email when it is their turn to play (set up an email server).

Proposed Solution

Network Part:

  • A router to interconnect the city with the rest of the world
  • An IP addressing plan
  • A backup of the configuration on a server
  • LAN traffic security

System Part:

  • A web server, hosting the website for displaying scores and launching games
  • A mail server and a webmail for appointment messages
  • DNS server for resolving DMZ and Trusted names
  • An Active Directory directory server to authenticate players

Match Management:

  • A secure web interface for displaying results (protected by directory login/password)
  • The ability to launch a server to host a competition (Administrator)
  • To prevent cheating, the player must have the game and only the game on their screen

What I learned from this project

  • Gained a better understanding of the agile SCRUM methodology in a project: it is a very efficient method, based on good communication among the team and daily meetings.
  • Deployed network services: it was the first project in which I used real hardware, as we usually simulate everything on Cisco Packet Tracer in class to learn the basic commands.
  • Developed a website: I've had experience in this field before, but this time I was completely in charge of implementing and updating the site. I learned more about the functioning of GitHub, beyond just "push" and "pull", understanding how branches work and their usefulness. I also enjoyed discovering dynamic animations on the web with JavaScript. This project is actually the reason why I decided to create this portfolio.
  • This project required knowing the basic commands, both in shell and Cisco IOS.
  • Increased knowledge: learned about backups (discovered Oxidized) and how easy it is to set up WiFi for a network.

Prerequisites

  • TCP/IP Networks
  • Cisco IOS
  • Linux Shell
  • Web
  • PHP
  • VirtualBox
  • Windows Server (An Active Directory)
  • Apache
